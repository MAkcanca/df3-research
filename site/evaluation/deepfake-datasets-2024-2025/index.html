    Latest Academic Deepfake Datasets (2024-2025)¶

    This document lists the newest academically-released deepfake datasets suitable for testing DF3, focusing on image-based datasets that complement the existing synthetic-only datasets (GenImage, DRAGON, Nano-banana).

    Top Recommendations¶

    1. MFFI (Multi-Dimensional Face Forgery Image Dataset) ⭐ RECOMMENDED¶

    Publication: 2025 (arXiv:2509.05592)

    Scale & Diversity: - ~1,024,000 images (real + fake) - 50 different forgery methods covering: - Face swapping - Face reenactment - Face synthesis - Face editing - High diversity in scenes, image quality, and realistic degradations - Closer to "in-the-wild" appearances than lab-generated datasets

    Why It's Good for DF3: - Massive scale allows for robust evaluation - Covers multiple manipulation types (not just generation) - Includes realistic compression/degradation artifacts - Well-suited for testing forensic tool effectiveness

    Access: - Check arXiv paper: Multi-Dimensional Face Forgery Image Dataset - Dataset likely hosted on academic repository or Hugging Face - May require research agreement/license

    2. DDL (Deepfake Detection & Localization Dataset)¶

    Publication: 2025 (arXiv:2506.23292)

    Scale & Features: - >1.4 million forged samples - >1.18M spatial masks (pixel-level annotations) - >0.23M temporal segments (for video) - 80 distinct deepfake methods - Covers 7 manipulation architectures, 7+ classic and 3 novel forgery modes - Supports both detection and localization tasks

    Why It's Good for DF3: - Largest scale dataset available - Includes spatial masks (useful if you want to test localization) - Covers both images and videos (can extract frames) - Extremely diverse in manipulation methods

    Access: - Check arXiv paper: DDL: A Comprehensive Dataset for Deepfake Detection and Localization - Dataset repository information should be in the paper

    3. DF40 (DeepfakeBench DF40)¶

    Publication: 2024-2025

    Scale & Diversity: - Tens of thousands of images - 40 different face forgery methods: - Face swapping - Face reenactment - Full-face synthesis - Face editing - Includes unknown/black-box methods (good for generalization testing)

    Why It's Good for DF3: - Broadest diversity in face forgery methods - Includes commercial/black-box tools (realistic scenario) - Well-curated and academically validated - Part of DeepfakeBench ecosystem

    Access: - Website: DF40 Dataset - Check DeepfakeBench repository for download links - May require academic/research agreement

    4. Deepfake-Eval-2024 (In-The-Wild Benchmark)¶

    Publication: 2025 (arXiv:2503.02857)

    Scale & Realism: - ~2,000 images (plus video/audio) - Collected from 88 websites and social media platforms - Content generated & circulated during 2024 (very current) - 52 languages represented - Real-world compression, artifacts, and degradation

    Why It's Good for DF3: - Tests robustness on real-world content (not lab-generated) - Shows how detectors perform on current deepfake techniques - Includes social media compression artifacts - Demonstrates performance drops vs. "legacy" academic datasets

    Access: - Check arXiv paper: Deepfake-Eval-2024: An In-The-Wild Deepfake Detection Benchmark - Dataset likely available via Hugging Face or academic repository - May require usage agreement

    5. SID-Set / SIDA (Social Media Image Detection Dataset)¶

    Publication: CVPR 2025

    Scale & Features: - ~300,000 images (real + AI-altered) - High realism and comprehensive annotations - Tasks: Detection, localization, and explanation - Includes mask predictions and textual justifications

    Why It's Good for DF3: - Social media focus (realistic use case) - Includes localization masks (if you want to test spatial detection) - Large scale - Well-annotated for research

    Access: - CVPR 2025 paper: SIDA: Social Media Image Deepfake Detection, Localization and Explanation - Check paper for dataset download links - May require research agreement

    How to Integrate with DF3¶

    Step 1: Download Dataset¶

    Follow the dataset's official download instructions (usually from the paper or repository).

    Step 2: Convert to DF3 Format¶

    DF3 expects a JSONL file with this format:

    {"id": "sample-001", "image": "path/to/image.jpg", "label": "fake"}
{"id": "sample-002", "image": "path/to/image.jpg", "label": "real"}

    Prevent Label Leakage in File Paths

    Critical: Ensure file paths and filenames do not reveal the label. LLMs can cheat by reading file paths (e.g., fake/image.jpg or deepfake_001.jpg) instead of analyzing image content. Use neutral identifiers like sample-001.jpg, image_123.jpg, or random hashes. Avoid paths containing words like "fake", "real", "deepfake", "synthetic", "authentic", etc.

    You can create a conversion script:

    import json
from pathlib import Path
import hashlib

# Example: Convert dataset directory structure to JSONL
# IMPORTANT: Use neutral identifiers to prevent label leakage
def convert_to_jsonl(dataset_dir, output_file):
    samples = []
    counter = 0

    # Real images - use neutral IDs, not "real-*"
    real_dir = Path(dataset_dir) / "real"
    for img_path in real_dir.glob("*.jpg"):
        # Use neutral identifier (counter or hash) instead of "real-*"
        neutral_id = f"sample-{counter:06d}"
        samples.append({
            "id": neutral_id,
            "image": str(img_path),  # Note: original path may contain "real/"
            "label": "real"
        })
        counter += 1

    # Fake images - use neutral IDs, not "fake-*"
    fake_dir = Path(dataset_dir) / "fake"
    for img_path in fake_dir.glob("*.jpg"):
        neutral_id = f"sample-{counter:06d}"
        samples.append({
            "id": neutral_id,
            "image": str(img_path),  # Note: original path may contain "fake/"
            "label": "fake"
        })
        counter += 1

    # Write JSONL
    with open(output_file, 'w') as f:
        for sample in samples:
            f.write(json.dumps(sample) + '\n')

# Better approach: Copy files to neutral directory structure first
def convert_to_jsonl_safe(dataset_dir, output_file, neutral_output_dir):
    """
    Copies images to neutral directory structure to prevent path-based cheating.
    """
    samples = []
    counter = 0
    neutral_output = Path(neutral_output_dir)
    neutral_output.mkdir(exist_ok=True)

    # Process real images
    real_dir = Path(dataset_dir) / "real"
    for img_path in real_dir.glob("*.jpg"):
        neutral_id = f"sample-{counter:06d}"
        neutral_path = neutral_output / f"{neutral_id}.jpg"
        # Copy to neutral location
        import shutil
        shutil.copy2(img_path, neutral_path)
        samples.append({
            "id": neutral_id,
            "image": str(neutral_path),  # Neutral path, no label info
            "label": "real"
        })
        counter += 1

    # Process fake images
    fake_dir = Path(dataset_dir) / "fake"
    for img_path in fake_dir.glob("*.jpg"):
        neutral_id = f"sample-{counter:06d}"
        neutral_path = neutral_output / f"{neutral_id}.jpg"
        shutil.copy2(img_path, neutral_path)
        samples.append({
            "id": neutral_id,
            "image": str(neutral_path),  # Neutral path, no label info
            "label": "fake"
        })
        counter += 1

    # Write JSONL
    with open(output_file, 'w') as f:
        for sample in samples:
            f.write(json.dumps(sample) + '\n')

convert_to_jsonl_safe("path/to/dataset", "data/deepfake_eval.jsonl", "data/neutral_images")

    Step 3: Run Evaluation¶

    python scripts/evaluate_llms.py \
    --dataset data/deepfake_eval.jsonl \
    --models gpt-5.1 \
    --tools both

    Recommendations by Use Case¶

    If you want the largest scale:¶

    → DDL or MFFI (>1M samples)

    If you want real-world robustness:¶

    → Deepfake-Eval-2024 (in-the-wild content)

    If you want method diversity:¶

    → DF40 (40 methods) or DDL (80 methods)

    If you want social media realism:¶

    → SID-Set (~300K images)

    If you want localization capability:¶

    → DDL or SID-Set (include spatial masks)

    Notes¶

    • Licensing: Most academic datasets require research agreements or have specific usage terms. Always check the dataset's license before use.
    • Storage: Large datasets (MFFI, DDL) may require significant storage space. Plan accordingly.
    • Sampling: For initial testing, consider sampling a subset (e.g., 500-2000 images) before running full-scale evaluation.
    • Comparison: Keep your existing synthetic dataset results separate for comparison—these new datasets test different aspects (manipulation vs. generation).
